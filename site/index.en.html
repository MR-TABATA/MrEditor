Zero is not a rounding. The file is mapped, never copied: with all 10 GB open, vmmap reports several GB resident (it varies run to run) and 0 bytes dirty for it — pages the OS can drop at any moment, holding nothing of ours. The app's own footprint is about 130 MB, and it stays about that whether or not the file is open: opening 10 GB adds only the page tables for the mapping, not your log. Measured 2026-07-15 on the shipping 1.3 build; 1.4 only adds small-file JSON handling and does not touch these paths.

The visual

27 seconds. No cuts.

One take, real speed: a 10 GB log opens, and we scroll and search it while the line index is still being built in the background. When the index lands, the status bar stops guessing — 86,420,337 lines, exactly — and ⌘L jumps to the last one.

The index — counting all 86 million lines — takes about 10 seconds on this file. Watch the status bar: it shows an estimate (about 89,292,800) and settles on the exact count when the scan finishes. The view never blocks while that runs. Reading, scrolling and searching stay live throughout; the search in this clip finds 137,111 matches while the index is at 95%.

How

The first line appears after reading a few dozen kilobytes.

Out of 10 GB. The rest is streamed in the background to build a line index, and none of it is kept in memory. Loading the whole document first — the usual approach — falls over at this size. So MrEditor does not do that.

mmap

Never load it all

The file is memory-mapped. The OS pages in only what you actually touch — so opening is near-instant at any size.

sparse index

~800 KB for 100 M lines

Keep one byte offset every 2,000 lines. Any line is a quick scan away. The whole index is tiny.

visible only

Draw a few dozen lines

A fixed-size view with a line-unit scroller — no 1.6-billion-point document view to blow past float precision.

Features

Read it. Then change it.

  • Compare / diff — two files, two open tabs, the clipboard, or a URL. Side by side, down to the character that changed
  • Merge — click → and the left side lands in the right. Save the result; the originals are never touched
  • Edit & save, any size — small files in memory, huge files via a piece table
  • Structured view — CSV/TSV columns, NDJSON fields & JSON pretty-print, millions of rows instantly
  • JSON query — a jmespath-style expression (⌥⌘J) filters & projects in place; the result is never saved
  • Atomic save — write to a temp file, then swap; the original is never left half-written
  • Save encoding — convert between UTF-8 / Shift-JIS / EUC-JP on save
  • Search — streams over mmap; multi-term AND, regex & case toggle
  • Filtered view / live grep — show only matching lines
  • Multiple documents — open many files, switch from a sidebar
  • Session restore — reopens your sidebar on launch, unsaved drafts included
  • Recent files — File ▸ Open Recent
  • Go to line & font zoom — ⌘L, ⌘+ / ⌘- / ⌘0
  • Follow mode — tail -f, auto-scroll as it grows
  • Copy & navigate — ⌘C, find next / previous
  • Encoding auto-detect — UTF-8 / Shift-JIS / EUC-JP
  • Color themes — System / Solarized / Monokai / custom; editor + UI
  • Fonts & display — family/size, tab width, line spacing, caret shape
  • Status bar — encoding, lines, size, index progress
  • Localized UI — English & Japanese
